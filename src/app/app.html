MOC Process Guide

Step-by-step process flow with role responsibilities and key rules.

1

Initiation

Role: MOC Owner

The Owner identifies a proposed change and initiates an MOC. Fills in title, description, basis, disciplines, target implementation date, and uploads a supporting document.

  • Select all relevant engineering disciplines
  • A confirmation popup appears before submission — once submitted it cannot be edited
  • System automatically routes to assigned Evaluators
2

Evaluation

Role: Evaluators (discipline-specific)

Each assigned Evaluator completes their discipline checklist and may raise action items. Evaluation action items assigned to the evaluator themselves must be closed before they can complete their evaluation.

  • Complete all required checklist items
  • Raise Evaluation, Pre-Startup, or Post-Startup action items as needed
  • Action items assigned to others only block the overall MOC from advancing — not the individual evaluator
  • All Evaluation-phase action items must be closed before the Owner can advance
3

Approval to Implement

Role: Owner → Manager

Once all evaluations and evaluation action items are complete, the Owner submits for approval. The Manager then reviews and decides.

  • Owner — clicks "Submit for Approval to Implement"
  • Manager — reviews evaluation results and approves OR rejects with a mandatory comment
  • If rejected, all evaluations are reset. Owner revises and resubmits
  • If cancelled by manager, the MOC is permanently closed
  • Routing tracker records all decisions with timestamps and comments
4

Implementation

Role: Owner

Once the Manager approves, the Owner must manually click "Begin Implementation" to start work. This gives the Owner full control over when physical work begins.

  • Begin Implementation — Owner confirms work has started
  • Mark Implementation Complete — Owner confirms all field work is done
  • The MOC then routes to PSSR
5

PSSR (Pre-Startup Safety Review)

Role: Operations Supervisor → Owner

Operations completes the PSSR checklist. Pre-Startup action items (raised earlier) must all be closed during this phase before PSSR can be submitted.

  • Pre-Startup action items are only completable during the PSSR step
  • Only the person assigned to the action item can complete it
  • Operations clicks "Submit PSSR for Review" — MOC stays in PSSR state with status "PSSR Completed"
  • Owner reviews and then manually clicks "Submit for Startup Approval" to advance
6

Approval for Start Up

Role: Manager → Owner

Manager reviews PSSR results and approves or rejects startup with mandatory comments.

  • Approve Start Up — Owner proceeds to Ready for Closure / Post-Startup Review
  • Reject — Send Back to PSSR — with mandatory reason, for additional review
  • All decisions are recorded in Approval History at the bottom of the MOC page
7

Ready for Closure / Post-Startup Review

Role: Assigned Action Item Owners → Owner

Post-Startup action items (raised during PSSR) must be completed by their assigned persons during this phase. Only then can the Owner close the MOC.

  • Post-Startup action items are only completable at this stage
  • Only the assigned person can mark their own action item complete
  • Completion requires a closure comment (and optionally an evidence file)
  • Owner closes the MOC once all post-startup items are done

Closed

Final State

The MOC is closed with a closure date, closing user, and full audit trail. All history is preserved in the routing tracker.

Key Rules & Guardrails

Assigned person only — Action items can only be completed by the person they are assigned to. No exceptions except Admin.
Stage enforcement — Pre-Startup items: PSSR step only. Post-Startup items: Ready for Closure step only. Evaluation items: Evaluation step only.
Mandatory comments — Manager rejections and cancellations require a written reason before the action button activates.
Full audit trail — All state changes, approvals, and rejections are recorded with timestamps and user identity in the Routing Tracker.
Owner controls advancement — At key gates (post-approval, post-PSSR completion) the Owner must manually advance the MOC. Nothing advances automatically.
