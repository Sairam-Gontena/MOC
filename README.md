# MOC Angular Prototype

Angular implementation of `FEATURE 9634531 - Step 1: Initiation`.

## Run

```powershell
npm install
npm run build
node serve-dist.js
```

Current local URL:

`http://127.0.0.1:4300/`

## Implemented Scope

- Owner landing page with `+ Create New MOC`.
- Summary cards: total MOCs, pending review, waiting on others, closed.
- Owner MOC table with MOC ID, title, workflow step, status, waiting on, action flag.
- Empty state with create action.
- Single-column initiation form.
- Required fields:
  - Description
  - Basis / Justification
  - Disciplines Involved
  - Supporting Document
  - Date Change Needs to be Implemented By
- Checkbox-style discipline selector with selected pills.
- File picker that stores file name only.
- Inline validation messages.
- Cancel action returning to landing.
- Double-submit guard.
- System-assigned fields displayed before submit.
- MOC ID autogenerated as `MOC000123` style.
- Submitted MOC moves to `Evaluation`.
- Redirects to read-only In-MOC view after submission.
- Read-only initiation data and workflow stepper.
- Dummy localStorage data only.
- Login page with demo users for Owner, Evaluator, Manager, and Admin.
- RBAC-filtered dashboards:
  - Owners see only MOCs they initiated.
  - Evaluators see only MOCs assigned to their discipline.
  - Managers see only MOCs awaiting their approval.
  - Admins see all MOCs.

## Explicit Non-Goals For This Step

- No evaluator assignment logic.
- No notifications.
- No risk screening or OE impact logic.
- No enterprise integrations.
- No PSSR/later phase artifacts.
